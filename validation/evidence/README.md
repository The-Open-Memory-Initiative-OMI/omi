# Validation Evidence Directory

## Purpose

This directory stores evidence artifacts generated during L0 validation runs.
Each run of `l0_runner.py` produces output files here.

## Expected Contents After L0 Execution

| File | Generated By | Description |
|------|-------------|-------------|
| `pinmap_report.json` | `verify_pinmap.py` | Pin map verification results (288/288) |
| `naming_report.json` | `verify_naming.py` | Naming consistency audit results |
| `lane_matrix.csv` | `verify_naming.py` | Byte-lane-to-pin mapping matrix |
| `l0_summary.json` | `l0_runner.py` | Unified L0 summary with hashes |
| `l0_summary.md` | `l0_runner.py` | Human-readable L0 summary |

## Generating Evidence

From the repository root:

```bash
python docs/08_validation_and_review/scripts/l0_runner.py \
  --reference design/connector/ddr4_udimm_288_pinmap.csv \
  --output-dir validation/evidence
```

## Hash Verification

Every evidence file includes a SHA-256 hash in `l0_summary.json`.
To verify independently:

```bash
# Linux / macOS
sha256sum validation/evidence/*.json validation/evidence/*.csv

# Windows (PowerShell)
Get-FileHash validation/evidence/*.json,validation/evidence/*.csv -Algorithm SHA256
```

## Storage Convention

For formal L0 submissions, copy evidence into a run-specific directory:

```
validation/results/RUN-YYYYMMDD-LOCAL-L0-001/
```

See `docs/08_validation_and_review/08-4_reporting_template.md` for naming rules.
